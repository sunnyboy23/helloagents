# stages

## 职责

定义工作流核心阶段（design/develop）规则，约束阶段入口、输出与验收。

## 接口定义（可选）

### 公共API
| 文件 | 说明 |
|------|------|
| stages/design.md | 方案设计阶段流程 |
| stages/develop.md | 开发实施阶段流程 |

## 行为规范

### 方案设计
**条件**: R2/R3 评估确认后进入
**行为**: 上下文收集 → 方案构思 → 方案包生成
**结果**: 产生 proposal.md 与 tasks.md

### 开发实施
**条件**: 方案包通过校验
**行为**: 逐任务执行 → 同步知识库 → 归档
**结果**: 方案包迁移至 archive
